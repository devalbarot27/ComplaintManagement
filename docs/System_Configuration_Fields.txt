SYSTEM CONFIGURATION - DATABASE FIELD LIST
==========================================

Database Name: complaint_management
Access: System Admin only (user_master.role = 6)
Migration: sql/migrate_system_configuration_masters.sql
Migration Runner: sql/run_system_configuration_migration.php

Shared API: api/system_config_datatable.php, api/system_config_get.php
Shared JS: js/system_config_master.js
Shared Helpers: includes/system_config_master_helpers.php


All five masters share the same table structure and CRUD pattern:
  - Add / Edit form with Name + Status
  - DataTable list with search and pagination
  - Soft delete (deleted_at)
  - Only active records appear in application dropdowns


COMMON status VALUES (all masters)
----------------------------------
active
inactive


COMMON TABLE COLUMNS (all masters)
----------------------------------
id                  SERIAL PRIMARY KEY
name                VARCHAR(100) NOT NULL      (Unique display/value name)
status              VARCHAR(20) NOT NULL       (active / inactive)
created_by          VARCHAR(100)               (Username who created the record)
created_at          TIMESTAMP NOT NULL         (Record creation timestamp)
updated_at          TIMESTAMP NULL             (Last update timestamp)
deleted_at          TIMESTAMP NULL             (Soft delete timestamp)


COMMON FORM FIELDS (Add / Edit - all masters)
---------------------------------------------
record_id           Hidden                     (0 = Add, >0 = Edit)
submit_*            Hidden                     (Form submit flag - see each module)
name                Text *                     -> {table}.name
status              Dropdown *                 -> {table}.status


COMMON LIST COLUMNS (DataTable - all masters)
---------------------------------------------
ID, Name, Status, Created At, Action

======================================================================
API PARAMETERS
======================================================================

system_config_datatable.php (POST)
----------------------------------
type                Required                   (industry_segment | warranty_chargeable |
                                               part_replaced | customer_feedback | reason)
draw, start, length  DataTables pagination
search[value]       Search text (matches name, status)
order               Sort columns: id, name, status, created_at


system_config_get.php (GET)
---------------------------
type                Required                   (master type key - see above)
id                  Required                   (Record ID for edit form prefill)


======================================================================
VALIDATION RULES (all masters)
======================================================================
- name is required
- name max length: 100 characters
- name must be unique among active (non-deleted) records (case-insensitive)
- status is required (active or inactive)
- Inactive options are hidden from dropdowns but existing transaction values remain valid

    -> Active names load into dropdowns on After Market forms
      -> Selected name is saved as text in the transaction table column
